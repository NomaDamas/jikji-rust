# Jikji Agent Routes

1. 먼저 `jikji find <root> "질문" --json`을 호출해 후보 목록과 evidence를 받는다.
2. 후보가 그럴듯하면 `answer_paths[]` 또는 `candidates[].p` 경로를 그대로 사용하고, 필요할 때만 `wiki`/`cache`를 읽는다.
3. 후보가 애매하면 `.jikji/graph_routes.jsonl`과 `.jikji/knowledge_graph.json`에서 term/intent/folder 이웃을 따라간다.
4. 자연어/내용 단서 후보는 `.jikji/file_cards.jsonl`와 `.jikji/chunk_map.jsonl`에서 보조 확인한다.
5. 폴더 후보는 `.jikji/folder_index.jsonl` 또는 `.jikji/folder_profile.jsonl`에서 찾는다.
6. 파일 후보는 `.jikji/file_index.jsonl`에서 찾는다.
7. 중복/백업 사본은 `.jikji/duplicate_map.jsonl`에서 확인한다.
8. PDF/Office/HWP 문서 본문은 `.jikji/doc_text/`에서 `rg`로 검색한다.
9. txt/md/csv/json/log 같은 텍스트형 파일은 원본 폴더에서 `.jikji`를 제외하고 검색한다.
10. 최종 접근은 `path` 필드의 원본 파일 경로를 사용한다.
11. 검색 작업 중 원본 파일을 이동/이름변경/삭제하지 않는다.

생성 시각: <TIMESTAMP>
캐시 정책: sha256, recomputed only when size or mtime_ns changed or cache is missing

직Jikji
Local file maps for AI agents

에이전트가 파일을 헤매지 않고 찾게 만든다

Jikji는 파일을 정리하지 않는다. 파일을 옮기지도 않는다. 대신 Claude Code, Codex, Hermes, OpenCode 같은 로컬 에이전트가 바로 사용할 수 있는 파일 지도와 즉시 검색 인덱스를 만든다.

Non-destructive No embeddings / No RAG core PDF · HWP · HWPX · Office text cache

As-is vs To-be

차이는 “에이전트가 직접 뒤지는가”와 “이미 만든 지도를 보고 후보를 받는가”다.

AS-IS

Raw agent 탐색

1
질문을 받음파일명·본문·폴더 기억이 섞여 있음
2
검색어를 추측키워드가 빗나가면 다시 시도
3
폴더를 순회검토본, 임시보관, 원본_섞임 사이를 헤맴
4
문서를 열어봄PDF/HWP/XLSX 본문 확인 비용이 반복됨
5
후보를 재판단중복·decoy·비슷한 파일에서 순위가 흔들림
느리고, 반복되고, 에이전트 턴을 많이 쓴다
TO-BE

Jikji 장착 탐색

1
명시된 루트 확인에이전트가 먼저 jikji brief 호출
2
후보를 즉시 받음파일명·경로·메타데이터·본문 인덱스 검색
3
근거를 함께 봄본문 스니펫, 폴더 맥락, matched terms 제공
4
필요할 때만 검증원본 파일은 마지막 확인 수단
5
정확한 경로 반환상대 경로를 그대로 답함
빠르고, 근거 있고, 재사용 가능한 탐색

Jikji가 만들어내는 실체

원본을 건드리지 않는 생성물

.jikji_agent_map.md루트에 보이는 에이전트용 시작 지도.
.jikji/search_index.sqliteEverything-style 사전 검색 인덱스.
.jikji/doc_text/PDF, HWP, HWPX, DOCX, PPTX, XLSX 등에서 추출한 본문 텍스트 캐시.
file_cards.jsonl파일별 경로, 확장자, 태그, 파싱 상태, 검색 단서.
folder_profile.jsonl폴더가 어떤 역할과 파일 구성을 갖는지 요약.
agent_routes.md후보 → 인덱스 → doc_text → 원본 순서의 안전한 탐색 경로.

에이전트가 호출하는 방식

git clone https://github.com/nomadamas/jikji.git
cd jikji
python3 -m venv .venv
.venv/bin/pip install -e .

jikji brief ~/Documents \
  "작년 봄 계약서 pdf" \
  --top-k 10 --json

{

Jikji는 파일 정리 앱이 아니다

Jikji는 로컬 에이전트가 사용자의 현재 파일·폴더·문서 내용을 더 잘 찾도록 만드는 비파괴 파일 지도 레이어다. 폴더 구조는 그대로 두고, 탐색 능력만 장착한다.

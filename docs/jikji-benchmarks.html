Jikji 벤치마크 결과 모음

Jikji는 로컬 폴더를 옮기거나 이름을 바꾸지 않고, 로컬 AI 에이전트가 파일을 찾도록 텍스트 우선 지도/인덱스를 만드는 비파괴 레이어다. 임베딩·벡터DB·RAG·LLM·리랭커를 쓰지 않고 결정론적 lexical/file-map 검색만 사용한다.

아래는 과거에 Jikji로 수행한 모든 벤치마크의 헤드라인 결과다. 두 종류의 증거가 있다: 결정론적 진단(검색 레이어만, 저비용 회귀 신호)과 실제 에이전트(actual-agent) 비교(Hermes/Claude, 워크스테이션·모델·실행에 따라 변동).

방법론 / 정직성. raw = Jikji 인덱스/파서 캐시를 쓰지 않고 원본 파일명·경로·native-text만 검색. jikji = Jikji 지도·인덱스·파서 텍스트 캐시 검색. Jikji 코어는 임베딩·벡터DB·RAG·LLM·리랭커 없이 결정론적 lexical/file-map만 사용한다. 실제 에이전트(Hermes/Claude) 타이밍은 워크스테이션·모델·실행마다 변동하므로, 대규모 결정론적 test를 주 회귀 신호로, 소규모 actual-agent run을 제품 sanity check로 사용한다. train/valid는 일반 개선 유도용으로만 쓰고 test 헤드라인과 분리했다.
